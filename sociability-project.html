Sociability

Home > Work > Sociability

Overview

Brief: Design an app aimed to help people find new things to do in their local area.


Instructions given in brief:

  • User base are students (18-24)
  • Base UI on existing popular UI designs

Role: UX / UI Designer & Interaction Design Lead

Duration: 1 Week

Wireframing

I joined the team creating Sociability as they were creating a user persona and user scenario, about to start the UI design phase. My job was to create the base functionality and interaction using the user persona and what already exists and is popular.

When beginning to design the UI, I wireframed multiple iterations of UI, with different forms of navigation and ways to discover what forms of entertainment are in the local area.

After testing each of the wireframes I created, I kept the most functional, and easy to understand elements of the designs. I had a basic user flow which I could use from usability testing, to gain insights on what worked for the designs and what was not working.

I designed a map of the user’s location, which users can walk around and find entertainment. There is also a discover page where there are cards displaying each form of entertainment close to the user.

Insights from User Testing

5 moderated usability tests were conducted. I observed the actions each user did while taking notes. Before each click I asked the user where they thought each click was going to bring them. This was to get a better idea of the conceptual model each user had of the product.


Snippit of insights gained;

  • Sturggled with “+” button (Participant 1, Participant 2, Participant 3)
  • Wondered why there was no feedback (Participant 1)
  • Wanted friends to be able to show interest in plans (Participant 1)
  • Looked for CTA to get an overview of entertainment (Participant 2, Participant 5)
  • Suggested being able to see friends on map (Participant 1)
  • Wanted to use search filter (Participant 4)

Style Tile & Components

A style tile was created to get a feel of the UI that was expected. From there, a colour palette was made, with text and components following.

The majority of the components game from the ‘Material Design 2’ web kit.

Icons were all taken from ‘The Noun Project’ to ensure cohesiveness.

Final UI Design

Home: The homepage displays an interactive map which can be navigated by walking around. There are also buttons on top of the map to act as a filter, so that the recommendations create a more tailored experience for the user.

Discover: The discover page has cards showing what entertainment is available in the local area. Users can also search if they already have somewhere in mind.

Overview: The overview page pops up if the user taps on a location. Images, a short bio on the location and information such as opening hours and useful links will be displayed.

Plan: The plan page is created when a plan is made or the user has been invited to a plan by one of their friends. It will have information such as a time and location for the plan.

Social: The Social pages show all of the plans that have been made as well as invites to plans and changes that have been made. The user also can make plans of their own.

Confirmation: The plan confirmation modal pops up after the user has created a plan and invites are about to be sent to invitees. There are options to view details, to make changes, and to confirm the plans.

Reflection

Reflection on the UX and UI design process for Sociability;

  • An agile team would have been useful for this project, as there was information drop-off during the hand-offs.
  • More user testing and prioritising testing more prototypes would have been beneficial in working out the navigation and ease of use.
  • Creating a larger design system with icons, images, etc, would have helped during the high-fidelity design phase.
